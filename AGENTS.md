# Guide for Blog.derrick.rw

The project is built with Next.js 15, TypeScript, Velite, and Tailwind CSS for an MDX static blog codebase.

## Project Overview

This is a static blog built with:

- **Next.js 15** with App Router
- **TypeScript** for type safety
- **Velite** for content management and MDX processing
- **Tailwind CSS** for styling
- **Shadcn/UI** for components
- **MDX** for blog content with interactive components

## Coding Conventions for AI Agents

### General TypeScript Guidelines

- **Always use TypeScript** for all new code
- Use strict type checking - no `any` types unless absolutely necessary
- Follow existing naming conventions in the codebase
- Add JSDoc comments for complex functions and components
- Use meaningful variable and function names

### React Component Standards

- Use **functional components** with React hooks
- Follow the existing component structure pattern
- Use proper TypeScript interfaces for props
- Keep components focused and single-purpose
- File naming: PascalCase for components (`PageHeader.tsx`)

### Next.js App Router Conventions

- Use `page.tsx` for route pages
- Use `layout.tsx` for layouts
- Follow the app directory structure
- Use proper metadata API for SEO
- Implement proper loading and error states

### Styling Guidelines

- Use **Tailwind CSS** utility classes for styling
- Follow utility-first approach
- Use Shadcn/UI components when available
- Custom CSS only when Tailwind utilities are insufficient
- Maintain responsive design principles

## File Creation Guidelines

### Creating New Blog Posts

1. Create new `.mdx` file in `src/content/blog/`
2. Use kebab-case naming: `descriptive-title.mdx`
3. Include complete frontmatter with all required fields
4. Add appropriate images to `public/images/blog/`
5. Test locally with `npm run velite:dev`

### Creating New Components

1. Create in appropriate directory under `src/components/`
2. Use TypeScript with proper interfaces
3. Follow existing component patterns
4. Export default and named exports as needed
5. Add to MDX components if needed for blog posts

### Creating New Pages

1. Follow Next.js App Router conventions
2. Create `page.tsx` in appropriate directory under `src/app/`
3. Include proper metadata for SEO
4. Use existing layout components
5. Ensure responsive design
